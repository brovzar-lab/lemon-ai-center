## Project Overview

Billy Rovzar's CEO command center is a full-stack, single-user productivity application integrating Gmail, Google Calendar, Notion, and Anthropic AI into a warm-dark editorial dashboard. The tech stack comprises a Vite + React 18 + TypeScript frontend, Express backend with TypeScript compiled to CommonJS, Firebase/Firestore for auth and data persistence, and Railway for deployment. The project emphasizes calm UI, warm branding (chocolate-brown base with lemon-gold accents), and an intelligent personal assistant experience (dual-voice "Jarvis" + "Billy" briefs). All code is type-strict, well-modularized, and follows consistent server response schemas.

## Phase 4 — Engineering Analysis

### 1. Type Safety — Grade: A

- **Strict mode** enabled in both `tsconfig.json` (frontend) and `server/tsconfig.json`
- All domain types defined in `shared/types.ts:1-225`; imported by both frontend and backend
- Component props explicitly typed; no implicit `any` in production code
- **Minor**: 6 `any` instances in frontend tests, 14 in backend (mostly Firestore deserialization + test mocks) — acceptable
- `server/routes/claude.ts:82`: `let tokens: any` before immediate typed assignment — minor

### 2. Error Handling — Grade: B+

- Standardized `{ error: { code, message, retryable } }` shape across all 18 route files
- 131 `try/catch` blocks across routes; consistent error codes (UNAUTHENTICATED, RATE_LIMITED, etc.)
- `server/lib/tokenCache.ts:26-49`: Promise deduplication to prevent thundering herd during token refresh
- **Issues**:
  - `server/routes/gmail.ts:48`: Silent `catch { /* skip malformed */ }` — could mask auth revocation
  - `server/lib/session.ts:14-39`: Mixed `.then().catch()` + `async/await` styles
  - `server/routes/claude.ts:429`: Logs `err?.status` only, not full stack
  - Some audit log calls not wrapped in catch (e.g., `server/routes/auth.ts:140`)

### 3. Security Posture — Grade: A-

- AES-256-GCM encryption for all Google refresh tokens (`server/lib/encryption.ts`)
- CSRF middleware on all write routes (`server/middleware/csrfCheck.ts`)
- Email-only auth allowlist enforced at `server/routes/auth.ts:98-103`
- Helmet CSP enabled in production (`server/index.ts:37`)
- Explicit CORS allowlist with regex patterns (`server/index.ts:39-55`)
- Per-session rate limiting; no token logging
- **Issue**: `csrfCheck.ts:7-8` compares against single `ALLOWED_ORIGIN` env var, but CORS allows multiple origins. If frontend at a Cloudflare tunnel, write requests may fail with 403 unless `ALLOWED_ORIGIN` matches exactly.

### 4. Code Organization — Grade: A

- Clear three-layer separation: frontend / backend / shared
- 15 Zustand stores, one per domain — no god objects
- Middleware pipeline ordered correctly: requireAuth → csrfCheck → rateLimit → handler
- Path aliases (`@/*`, `@shared/*`) consistent across Vite and TypeScript configs
- No circular dependencies detected
- All CLAUDE.md conventions followed (stores, error shape, auth middleware, cookie config)

### 5. Dead Code & Tech Debt — Grade: B+

- `server/routes/notion.ts` exists but is never mounted (`server/index.ts:16` commented out with note)
- 4 TODO comments in codebase (all minor, v1 scope clarifications):
  - `src/components/WrapupCard.tsx:8`
  - `src/components/InboxSummary.tsx:11`
- 12 `console.log` statements in `server/lib/` (intentional init logging — acceptable)
- `CLAUDE.md` section 10 explicitly documents what is out of v1 scope (subtasks, drag-drop, mobile PWA)

### 6. Test Coverage — Grade: B

- **36 test files** total: 23 frontend (`src/__tests__/`), 13 backend (`server/routes/` + `server/lib/`)
- Critical paths tested: auth flow, Gmail threads, Calendar events, Claude brief gen, all middleware, encryption
- **Gaps**:
  - `server/lib/brain/` — no test files (semantic search over vault)
  - `server/routes/voice.ts` — no test file
  - `src/lib/firestore.ts` — Firestore subscription logic untested
  - Error/failure paths largely untested (happy-path bias)
  - No integration/E2E tests (auth → brief → email action)

### 7. Build & Deployment — Grade: A

- `npm run build`: `vite build` → `tsc -p server/tsconfig.json` → artifacts in `dist/` + `server/dist/`
- `npm start`: serves static `dist/` + Express SPA fallback + all API routes
- `railway.json`: Nixpacks, health check `/health`, restart ON_FAILURE (max 3)
- Firestore-backed sessions survive server restart
- `__Host-` cookie prefix not used (Cloudflare terminates TLS; server sees HTTP internally — documented tradeoff)
- **Issue**: `npm run build` does NOT invoke `tsc --noEmit` before compile. Type errors could silently reach production. `npm run typecheck` is a separate manual step.

### 8. Dependencies — Grade: A-

- 37 production + 22 dev dependencies; all current-generation (2024–2025)
- `package-lock.json` present; no high-severity vulnerabilities
- Native Node.js `crypto` used for encryption (no third-party crypto library)
- 4 deprecation notices in lock file (`supertest` upgrade suggestion, DOMException polyfill) — test-only, no prod risk
